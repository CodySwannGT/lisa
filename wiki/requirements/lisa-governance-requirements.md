# Lisa Governance Requirements

Lisa's durable product requirements center on making AI-assisted development repeatable, governed, and verifiable.

## Governance

- AI sessions should receive consistent project rules and coding expectations.
- Workflow commands should route common tasks into repeatable flows.
- Quality gates should block unsafe or unverified changes before commit, push, or CI completion.
- Templates should install and update downstream projects predictably.
- Verification should be empirical, with proof commands attached to meaningful work.

## Developer Experience

- Implementation teams should be able to use simple commands and plain-English requests.
- Platform experts should be able to codify standards once and distribute them through Lisa.
- Downstream projects should retain customization where Lisa uses create-only or merge strategies.

## Automation

- GitHub workflows should support CI, release, nightly improvement jobs, review handling, and auto-fix loops.
- The CLI should support setup, update, and project detection across supported stack families.
